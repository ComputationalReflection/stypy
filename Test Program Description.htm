Test suites

This section describes the different test types we used to test stypy.

  1. Stypy model testing
  2. Code generation testing
  3. Real Programs
  4. Usage of the Numpy library features

Stypy model testing

Stypy works by generating a Python program from any Python program it processes. This new Python program calculates the types of the original one. To do this, certain assumptions and concepts must be implemented: Contexts for function calls, SSA algorithm to calculate possible types when dealing with conditionals, dealing with data structures...This test block checks that these concepts are correctly implemented and test them using direct calls to their methods, simulating the calls that the code of the generated type-inference programs will do to them.

Stypy code generation testing

One of the most important parts of Stypy is the ability to generate a new Python program from the ones it processes. To achieve this, Stypy implements a Python 3-way code generator able to generate type-inference code from any Python syntactic construction. This test group checks this code generator by using small programs that use different Python constructs (calls, assignments, generators,...), so we can check if the generated code correctly checks the different Python types used in these constructs.

Programs

This group is composed by applications that are created for a specific and realistic purpose: perform scientific calculations, popular problem solving, data manipulation and parsing, etc. These programs serve much different purposes and have different sizes, due to the different complexity required by each solved problem. The programs in this group are not specifically designed as benchmarks, but as Python applications that, independently of its size, were designed with a real purpose in mind. We used stypy to check if the code of these programs has potential runtime errors.

To obtain programs that fit in this category we used the program collection provided by the Shed Skin Python compiler distribution. Shed Skin (http://code.google.com/p/shedskin/) is an experimental compiler that can translate pure, but implicitly statically typed Python programs into optimized C++. Although this distribution is promising, it lacks support for certain Python programming language features at this moment, so we decided not to include it yet in our measurements. However, the example library it provides (http://code.google.com/p/shedskin/downloads/list ) is excellent, and suits to our purposes.

Large Scale Applications

The final group of tests uses different numpy features to check if code generation of type-inference programs works well when integrating this library new functions and types. To do that, we used a collection of code fragments using these features from the official Numpy documentation, that presents code snippets from real applications using these features.
